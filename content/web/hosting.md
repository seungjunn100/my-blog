## 호스팅(Hosting)

호스팅은 웹사이트 또는 애플리케이션을 인터넷에서 접근 가능하도록 서버에 저장하고 제공하는 서비스다.

개발자가 만든 웹사이트는 기본적으로 로컬 컴퓨터에 존재하므로 외부에서 접근할 수 없고, 24시간 켜져 있지도 않다. 반면, 서버는 항상 켜져 있는 특징이 있다. 따라서 웹사이트를 서비스하려면 인터넷에 연결된 서버 컴퓨터에 파일들을 올려야 한다.

즉, 사용자가 브라우저를 통해 요청을 보냈을 때 그 요청에 응답할 수 있는 파일이 위치한 곳이 서버이며, 이 서버 공간을 제공하는게 호스팅이다.

### 호스팅의 핵심 역할

호스팅은 단순히 파일을 저장하는 것이 아니라, 다음과 같은 역할을 수행한다.

#### 파일 저장 및 제공

HTML, CSS, JS, 이미지 파일 등을 저장하고 사용자가 브라우저에서 요청을 보내면 해당 파일을 반환한다.

#### 서버 실행 환경 제공

사용자의 요청(로그인, 검색 등)을 처리할 수 있는 컴퓨팅 엔진(CPU, 메모리 등)이 돌아가는 환경을 제공하여 프로그램이 실제로 동작하게 만든다.

#### 데이터베이스 연결

많은 서비스들은 사용자의 정보, 게시글 등 데이터를 다룬다. 호스팅은 이런 데이터들을 자체 제공하기도 하지만 데이터의 양이 방대해지면 호스팅 서버와 데이터베이스 서버를 분리해서 사용할 수도 있다. 이 데이터베이스와 프로그램이 원활하게 대화할 수 있도록 연결하고 관리한다.

#### 네트워크 및 보안 처리

호스팅 서비스는 아래와 같은 인프라 관리를 대신해준다.

- 보안 통신 (HTTPS/SSL)
  
  데이터 전송 시 암호화를 적용하는 SSL 인증서 설정을 관리하여 사용자 정보를 안전하게 보호해준다.

- 접근 제어 (방화벽)
  
  허가되지 않은 비정상적인 접근을 차단하여 서버 내부의 데이터를 안전하게 지켜준다.

- 공격 방어 (DDoS 방어)
  
  서버를 마비시키기 위해 대량의 가짜 트래픽을 보내는 DDoS 공격을 감지하고 차단한다.

- 데이터 흐름 관리 (트래픽 처리)
  
  사용자가 몰릴 때 데이터 전송이 원활하게 이루어지도록 네트워크 대역폭을 조절하고 최적화한다.

### 호스팅 종류

#### 웹호스팅 (공유 호스팅)

여러 웹사이트가 하나의 서버 자원(컴퓨팅 엔진(CPU, 메모리 등))을 함께 사용하는 방식이다.

- 장점 : 비용이 매우 저렴하고 초보자도 쉽게 사용할 수 있다.

- 단점 : 다른 사이트의 트래픽 급증 시 내 사이트도 느려질 수 있다.

- 적합한 경우 : 소규모 블로그, 개인 포트폴리오, 초기 프로젝트

#### 전용 호스팅 (서버 호스팅)

서버 전체를 단독으로 사용하는 방식이다.

- 장점 : 성능과 보안이 안정적이며, 서버 설정을 자유롭게 구성할 수 있다.

- 단점 : 비용이 상대적으로 높고 직접 유지보수를 해야 한다.

- 적합한 경우 : 백엔드 서버 운영, 대규모 트래픽 서비스, 금융·의료 등 높은 보안 요구 서비스

#### 클라우드 호스팅

클라우드 호스팅은 여러 서버를 하나의 인프라처럼 사용하여 유연하게 확장 가능한 방식이다.

- 장점 : 트래픽에 따라 자원을 유연하게 확장/축소 가능하다.

- 단점 : 사용량에 따라 비용 예측이 어려울 수 있다.

- 적합한 경우 : 트래픽 변동이 큰 서비스, 스타트업, 엔터프라이즈

#### 서버리스 호스팅

서버리스 호스팅이란 서버를 직접 관리하지 않고, 요청이 발생할 때만 함수 단위로 코드를 실행하는 호스팅 방식이다.

- 장점 : 배포가 매우 간편하며, 서버 관리 및 설정 등 복잡한 인프라 지식 없이도 바로 서비스를 운영할 수 있다.

- 단점 : 오랫동안 호출되지 않다가 다시 실행될 때 발생하는 약간의 지연 시간으로 인한 초기 지연이 발생할 수 있고, 실행 시간 제한 및 서버 환경에 대한 제어가 제한적이다.

- 적합한 경우 : Next.js 같은 프레임워크 기반 서비스, 간단한 API 서버, 이벤트 기반 처리
